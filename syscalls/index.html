01 Códigos de exceção — registrador Cause ($13)

Todo evento excepcional — inclusive syscalls — é tratado pelo mesmo mecanismo. O campo ExcCode (bits 6–2) do Cause diferencia os casos pelo código numérico.

0
Interrupção externa
Dispositivo de E/S sinalizou evento (assíncrona)
4
AdEL — erro de endereço
Endereço inválido ou desalinhado em load
5
AdES — erro de endereço
Endereço inválido ou desalinhado em store
8
Syscall ← este!
Instrução syscall — pedido de serviço ao SO
10
RI — instrução reservada
Opcode inválido ou não implementado
12
Ov — overflow aritmético
Resultado de add/sub não cabe em 32 bits
13
Tr — trap
Instrução teq/tne com condição verdadeira
Regra prática: se o código for 8, não é erro — é pedido de serviço. O SO lê $v0 e retorna normalmente via eret. Em todos os outros casos de erro, o MARS encerra o programa.
02 Registradores do Coprocessador 0

O Coprocessador 0 é o núcleo de controle do MIPS — responsável por gerenciar exceções, modo de execução e estado do processador. Acessível apenas via mfc0 / mtc0.

Reg. Nº Nome Função Momento de uso
$8 8 BadVAddr Endereço que causou erro de acesso à memória (AdEL / AdES) após exceção de endereço
$12 12 Status Modo de execução (KUc bit 1), interrupções (IEc bit 0), pilha KU/IE de 3 níveis hardware altera a cada exceção
$13 13 Cause Código do tipo de exceção (bits 6–2). Lido pelo handler para identificar a causa imediatamente após exceção
$14 14 EPC Endereço da instrução após a exceção. Usado pelo eret para retornar ao programa hardware salva; eret restaura
$15 15 PRId Identificador do processador (fabricante e versão). Somente leitura identificação do hardware
STATUS ($12) — bits relevantes e pilha de 3 níveis

1KUc

BitsCampoSignificado
1KUc0 = modo kernel (atual)  ·  1 = modo usuário. Hardware zera ao entrar na exceção; eret restaura para 1
0IEc0 = interrupções desabilitadas  ·  1 = habilitadas. Hardware zera para proteger o handler de novas exceções

5–4KUp/IEpEstado mais antigo — terceiro nível da pilha. Uma terceira exceção sobrescreve este valor; ele é perdido
03 Endereço do handler e instrução eret
ENDEREÇO FIXO DO EXCEPTION HANDLER
0x80000180
Endereço fixo em hardware para onde o PC é desviado a cada exceção. O SO carrega o handler aqui durante a inicialização.
O endereço fica no espaço kseg1 — acessível apenas em modo kernel, não cacheável. Um programa de usuário não pode ler nem escrever nesse endereço.
INSTRUÇÃO eret — exception return
CampoValor
Sintaxeeret
OperaçãoPC ← EPC  ·  KUc ← 1  ·  IEc ← 1
EfeitoRestaura modo usuário e retorna ao programa
Quem usaExclusivamente o kernel
AtomicidadeAs três operações ocorrem em um único ciclo
Por que não $ra? $ra é acessível pelo programa de usuário — o SO não pode confiar nele. O EPC fica no Coprocessador 0, protegido, só acessível pelo kernel.
04 Instruções mfc0 e mtc0 — acesso ao Coprocessador 0
Instrução Operação Descrição
mfc0 $rt, $rd $rt ← Coproc0[$rd] Move From Coprocessor 0 — copia registrador do Coproc0 para registrador inteiro da CPU
mtc0 $rt, $rd Coproc0[$rd] ← $rt Move To Coprocessor 0 — copia registrador inteiro para registrador do Coproc0
identificar causa da exceção
# início do exception handler
mfc0 $k0, $13       # lê Cause → $k0
andi $k0, $k0, 0x3C  # isola bits 5–2
srl  $k0, $k0, 2     # desloca → código
# $k0 == 8 → era syscall
# $k0 == 0 → era interrupção
desabilitar interrupções no handler
# protege o handler de novas exceções
mfc0 $k0, $12        # lê Status
ori  $k1, $k0, 0x1   # prepara máscara
xori $k1, $k1, 0x1   # zera bit IEc
mtc0 $k1, $12        # grava Status
05 Fluxo completo — quem faz o quê em cada etapa
# Quem age Ação Detalhe
1 programa Prepara registradores li $v0, N — código do serviço  ·  $a0–$a3 — argumentos
2 programa Executa syscall Dispara exceção interna código 8
3 hardware Salva EPC EPC ← PC + 4 — guarda o endereço de retorno
4 hardware Grava Cause e Status Cause ← 8  ·  zera KUc e IEc no Status
5 hardware Desvia PC PC ← 0x80000180 — entra no exception handler
6 kernel Lê Cause mfc0 $k0, $13 — confirma código 8 (é syscall, não erro)
7 kernel Despacha por $v0 Lê $v0 e executa a rotina interna correspondente
8 kernel Executa o serviço Usa $a0–$a3 como argumentos; resultado em $v0 se houver
9 kernel Executa eret PC ← EPC  ·  restaura KUc = 1 e IEc = 1 — volta ao programa
06 Serviços disponíveis no MARS — tabela completa

Carregue o código do serviço em $v0 e os argumentos nos registradores indicados antes de executar syscall. Resultados ficam disponíveis após o retorno do SO.

$v0 Serviço Parâmetros de entrada Resultado / observação
Saída
1 print_int $a0 = inteiro (32 bits) Imprime o inteiro no console
2 print_float $f12 = float (precisão simples) Imprime o float no console
3 print_double $f12 = double (precisão dupla) Imprime o double no console
4 print_string $a0 = endereço da string Imprime string terminada em \0
11 print_char $a0 = código ASCII do caractere Imprime um único caractere
Entrada
5 read_int — $v0 = inteiro lido do teclado
6 read_float — $f0 = float lido do teclado
7 read_double — $f0 = double lido do teclado
8 read_string $a0 = buffer  ·  $a1 = tamanho máx. Lê string do teclado (máx. $a1−1 chars + \0)
12 read_char — $v0 = código ASCII do caractere lido
Memória e controle
9 sbrk $a0 = bytes a alocar no heap $v0 = endereço do bloco alocado
10 exit — Encerra o programa normalmente
17 exit2 $a0 = código de saída Encerra com código de retorno específico
Arquivo (MARS estendido)
13 open $a0 = path  ·  $a1 = flags  ·  $a2 = modo $v0 = descritor (−1 se erro)
14 read $a0 = fd  ·  $a1 = buffer  ·  $a2 = bytes $v0 = bytes lidos (−1 se erro)
15 write $a0 = fd  ·  $a1 = buffer  ·  $a2 = bytes $v0 = bytes escritos (−1 se erro)
16 close $a0 = fd Fecha o descritor de arquivo
Dica MARS: para serviços de leitura (5, 6, 7, 8, 12), ative Settings → Popup dialog for input syscalls para que o MARS abra uma janela de entrada em vez de exigir digitação no console integrado.

 4.text

11    li  $v0, 10   # exit

ex03 — read_int · lê e imprime inteiro
 1.data
 2p1: .asciiz "Digite um numero: "
 3p2: .asciiz "Voce digitou: "

 5.text
 6.globl main
 7main:
 8    la  $a0, p1  # prompt
 9    li  $v0, 4   # print_string
10    syscall

12    li  $v0, 5   # read_int → $v0
13    syscall

16    la  $a0, p2  # resposta

18    syscall

20    move $a0,$s0 # recupera valor

22    syscall

24    li  $v0, 10  # exit
25    syscall
ex04 — read_string · lê e ecoa string
 1.data
 2prompt: .asciiz "Seu nome: "
 3saudac: .asciiz "Ola, "
 4buf:    .space  32  # buffer 32 bytes

 6.text
 7.globl main
 8main:
 9    la  $a0, prompt # prompt
10    li  $v0, 4      # print_string
11    syscall

13    la  $a0, buf    # buffer destino
14    li  $a1, 32     # tamanho máximo
15    li  $v0, 8      # read_string
16    syscall

18    la  $a0, saudac # "Ola, "
19    li  $v0, 4      # print_string
20    syscall

22    la  $a0, buf    # imprime nome
23    li  $v0, 4      # print_string
24    syscall

26    li  $v0, 10     # exit
27    syscall
ex05 — soma de vetor · loop + print_int
 1.data
 2vet:  .word   10, 20, 30, 40, 50   # vetor de 5 inteiros
 3tam:  .word   5                    # tamanho do vetor
 4msg:  .asciiz "Soma: "

 6.text
 7.globl main
 8main:
 9    la   $t0, vet       # $t0 = ponteiro para vet[0]
10    lw   $t1, tam        # $t1 = tamanho
11    li   $t2, 0          # $t2 = acumulador (soma = 0)
12    li   $t3, 0          # $t3 = índice i = 0

14loop:
15    bge  $t3, $t1, fim  # se i >= tam → termina
16    lw   $t4, 0($t0)     # $t4 = vet[i]
17    add  $t2, $t2, $t4  # soma += vet[i]
18    addi $t0, $t0, 4   # avança ponteiro 4 bytes
19    addi $t3, $t3, 1   # i++
20    j    loop

22fim:
23    la   $a0, msg        # imprime "Soma: "
24    li   $v0, 4
25    syscall

27    move $a0, $t2       # imprime resultado
28    li   $v0, 1          # print_int
29    syscall

31    li   $v0, 10         # exit
32    syscall
