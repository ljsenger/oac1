01Visão geral das instruções de decisão

Além de instruções aritméticas e de movimentação de dados, um processador deve prover instruções que permitem alterar o fluxo de execução com base nos dados. Em alto nível, correspondem a if/else e laços while/for.

beq
beq $t1, $t2, endereço
Branch if Equal
Desvia para endereço se $t1 == $t2. Caso contrário, execução continua normalmente na próxima linha.
condicional
bne
bne $t1, $t2, endereço
Branch if Not Equal
Desvia para endereço se $t1 ≠ $t2. Útil para laços e condicionais invertidos.
condicional
j
j endereço
Jump (incondicional)
Sempre desvia para endereço. Usado para implementar o bloco else e saída de laços.
incondicional
slt
slt $rd, $rs, $rt
Set if Less Than
$rd ← 1 se $rs < $rt, senão $rd ← 0. Não desvia — apenas marca o resultado para uso posterior.
comparação
slti
slti $rt, $rs, imm
Set if Less Than Immediate
$rt ← 1 se $rs < imm (valor imediato de 16 bits com sinal), senão $rt ← 0.
comparação
jr
jr $rs
Jump Register
Desvia para o endereço contido no registrador $rs. Usado no retorno de subprogramas.
incondicional
02Desvios condicionais e incondicionais

O endereço de desvio em beq e bne é codificado como deslocamento relativo ao PC (16 bits, em words). O endereço em j é absoluto (26 bits). O programador escreve rótulos — o montador calcula os valores numéricos.

Instrução Condição de desvio Endereço Uso típico
beq $t1, $t2, rot$t1 == $t2relativo ao PCif/else, condição de igualdade
bne $t1, $t2, rot$t1 ≠ $t2relativo ao PClaços do-while, negação de condição
j endereçosempreabsoluto (26 bits)pular bloco else, sair de laço
jr $rssempreconteúdo de $rsretorno de subprograma
EXEMPLO — tradução de if/else com beq
fragmento em C
if ( a == 0 ) {
    a = b + 2;
} else {
    a = b + 1;
}
assembly MIPS
 1lw   $t0, 0($zero)     # a → $t0
 2lw   $t1, 4($zero)     # b → $t1
 3beq  $t0, $zero, entao  # a==0 → entao
 4addi $t0, $t1, 1       # else: a=b+1
 5j    saida              # pula bloco if
 6entao:
 7addi $t0, $t1, 2       # if:   a=b+2
 8saida:
 9sw   $t0, 0($zero)     # salva a
Dois fluxos possíveis: (a) se beq for verdadeiro → executa linha 7 (if). (b) se falso → executa linhas 4–5 (else) e o j saida garante que o bloco if não seja executado também.
03Instruções de comparação — slt e slti

beq e bne só testam igualdade. Para comparações de menor/maior, o MIPS usa slt/slti para marcar o resultado em um registrador, seguido de beq ou bne para tomar a decisão.

SLT — comparação entre registradores
CampoValor
Sintaxeslt $rd, $rs, $rt
Operação$rd ← ($rs < $rt) ? 1 : 0
TipoFormato R
Desvio?Não — apenas marca o resultado
SLTI — comparação com imediato
CampoValor
Sintaxeslti $rt, $rs, imm
Operação$rt ← ($rs < imm) ? 1 : 0
TipoFormato I (16 bits com sinal)
Desvio?Não — apenas marca o resultado
EXEMPLO — if $t2 < 32 then $t3 = 5
fragmento em C
if ( t2 < 32 ) {
    t3 = 5;
}
assembly MIPS — padrão slti + beq
 1slti $t4, $t2, 32     # $t4=1 se $t2<32
 2beq  $t4, $zero, saida  # $t4==0 → pula
 3addi $t3, $zero, 5    # $t3 = 5
 4saida: # continua...
Padrão slt + beq/bne: como o MIPS não tem instruções de desvio para <, >, ≤ ou ≥, o padrão é: (1) slt/slti coloca 0 ou 1 em registrador temporário; (2) beq ou bne toma a decisão com base nesse valor.
04Pseudo-instruções de comparação e desvio

O montador MARS disponibiliza pseudo-instruções que combinam internamente slt + beq/bne. Cada pseudo-instrução (exceto beqz) é expandida em duas instruções reais pelo montador.

Comparação em C Pseudo-instrução MIPS Instruções reais geradas pelo montador
a == bbeq $t0, $t1, entaoinstrução real — 1 instrução
a != bbne $t0, $t1, entaoinstrução real — 1 instrução
a < bblt $t0, $t1, entaoslt $at, $t0, $t1  ·  bne $at, $zero, entao
a > bbgt $t0, $t1, entaoslt $at, $t1, $t0  ·  bne $at, $zero, entao
a <= bble $t0, $t1, entaoslt $at, $t1, $t0  ·  beq $at, $zero, entao
a >= bbge $t0, $t1, entaoslt $at, $t0, $t1  ·  beq $at, $zero, entao
a == 0beqz $t0, entaobeq $t0, $zero, entao  (1 instrução real)
Registrador $at ($1): o montador usa $at como temporário interno para expandir pseudo-instruções. Nunca use $at em seu código — ele pode ser sobrescrito pelo montador sem aviso.
05Exemplos completos comentados

Programas prontos para testar no MARS. Convenção: f, g → $t0, $t1  ·  h → $t2.

ex01 — if/else com beq (a == b)
 1.data
 2a: .word 0
 3b: .word 7
 4.text
 5.globl main
 6main:
 7    lw  $t0, a
 8    lw  $t1, b
 9    beq $t0,$zero,entao  # a==0?
10    addi $t0,$t1,1       # else: a=b+1
11    j   saida
12entao:
13    addi $t0,$t1,2       # if:   a=b+2
14saida:
15    sw  $t0, a
16    move $a0,$t0
17    li  $v0,1 ; syscall    # imprime a
18    li  $v0,10 ; syscall
ex02 — if/else com bgt (f > g)
 1.data
 2f: .word 10
 3g: .word 8
 4h: .word 0
 5.text
 6.globl main
 7main:
 8    lw  $t0, f
 9    lw  $t1, g
10    bgt $t0,$t1,entao    # f>g? (pseudo)
11    move $t2,$t1         # else: h=g
12    j   saida
13entao:
14    sub $t2,$t0,$t1     # if: h=f-g
15saida:
16    sw  $t2, h
17    move $a0,$t2
18    li  $v0,1 ; syscall    # imprime h
19    li  $v0,10 ; syscall
ex03 — laço while com bge
 1.data
 2n: .word 5       # conta até 5
 3.text
 4.globl main
 5main:
 6    lw  $t0, n     # $t0 = n
 7    li  $t1, 0     # i = 0
 8loop:
 9    bge $t1,$t0,fim # i>=n → sai
10    move $a0,$t1
11    li  $v0,1 ; syscall # print i
12    addi $t1,$t1,1 # i++
13    j   loop
14fim:
15    li  $v0,10 ; syscall
ex04 — detectar negativo via srl
 1.data
 2x: .word -13   # valor a testar
 3a: .word 0
 4.text
 5.globl main
 6main:
 7    lw   $t0, x
 8    srl  $t1,$t0,31  # bit sinal→$t1
 9    bne  $t1,$zero,neg # 1→negativo
10    li   $t2, 10     # positivo: a=10
11    j    fim
12neg:
13    li   $t2, 5      # negativo: a=5
14fim:
15    sw   $t2, a
16    move $a0,$t2
17    li   $v0,1 ; syscall
18    li   $v0,10 ; syscall
06Exercícios
Exercício 6.1 — relacione os termos

Itens: (1) instrução que sempre modifica o PC incondicionalmente   (2) desvia se valores são iguais   (3) rótulo   (4) slt   (5) j fim   (6) slti   (7) desvie para saída se $t0 < $t1   (8) próxima instrução a ser executada

Preencha as lacunas com o item mais adequado:

lacunas
( ) set if less than — marca 1 se menor que
( ) PC (program counter)
( ) j
( ) comparação com valor imediato
( ) marca endereço de memória
( ) slt $at,$t0,$t1 seguido de bne $at,$zero,saida
( ) beq
( ) salto incondicional para o rótulo fim
Exercício 6.2 — traduzir if/else com igualdade

Crie um programa assembly MIPS equivalente ao fragmento C. Use: f: .word 10, g: .word 8, h: .word 5.

fragmento C
if ( f == g ) {
    h = h + 5;
} else {
    h = h - 1;
}
Exercício 6.3 — traduzir if/else com maior que

Crie um programa assembly MIPS equivalente ao fragmento C. Use: f: .word 10, g: .word 8, h: .word 0. Dica: use pseudo-instrução bgt ou o padrão slt + bne.

fragmento C
if ( f > g ) {
    h = f - g;
} else {
    h = g;
}
Exercício 6.4 — detectar número negativo

Crie um programa Assembly MIPS que detecte se o valor em memória é negativo usando srl e beq ou bne. Use o esqueleto: x: .word -13, a: .word 0.

Se negativo → armazene 5 em a. Se positivo → armazene 10.

Dica: em complemento de 2, o bit 31 é o sinal. srl $t1, $t0, 31 move o bit de sinal para $t1. Se $t1 == 1, o número é negativo.
